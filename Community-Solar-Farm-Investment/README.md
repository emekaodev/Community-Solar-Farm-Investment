# Community Solar Farm Investment Smart Contract

A decentralized platform for fractional ownership of renewable energy projects built on the Stacks blockchain using Clarity smart contracts.

## Overview

This smart contract enables community members to invest in solar farm projects through fractional ownership. Investors can purchase shares in solar projects and earn rewards based on the energy production and revenue generated by these renewable energy installations.

## Features

- **Fractional Ownership**: Purchase shares in solar farm projects with flexible investment amounts
- **Revenue Sharing**: Earn rewards based on energy production and your ownership percentage
- **Project Management**: Create and manage multiple solar farm projects
- **Share Trading**: Transfer ownership shares between users
- **Transparent Reporting**: Track energy production and financial performance
- **Reward System**: Automated calculation and distribution of earnings

## Contract Architecture

### Core Components

#### Data Structures

- **solar-projects**: Stores project information including capacity, pricing, and status
- **user-shares**: Tracks individual user ownership and investment details  
- **project-energy-production**: Records energy output and revenue data
- **user-rewards**: Manages earnings and reward distributions

#### Key Functions

##### Public Functions

- `create-project`: Create a new solar farm investment opportunity
- `purchase-shares`: Buy fractional ownership in a project
- `record-energy-production`: Log energy output and revenue (project owners only)
- `calculate-rewards`: Compute user earnings based on ownership percentage
- `claim-rewards`: Withdraw accumulated earnings
- `transfer-shares`: Trade ownership between users
- `deactivate-project`: Admin function to disable a project

##### Read-Only Functions  

- `get-project`: Retrieve project details
- `get-user-shares`: Check user ownership in a project
- `get-user-rewards`: View earnings and pending rewards
- `get-energy-production`: Access production data
- `get-contract-info`: Get contract metadata

## Usage Guide

### For Project Creators

1. **Create Project**: Call `create-project` with project details including name, location, capacity, share price, and energy rate
2. **Record Production**: Regularly update energy production data using `record-energy-production`
3. **Monitor Performance**: Track project metrics and investor participation

### For Investors

1. **Browse Projects**: Use `get-project` to explore available investment opportunities
2. **Purchase Shares**: Invest using `purchase-shares` with desired number of shares
3. **Track Earnings**: Monitor rewards with `get-user-rewards`
4. **Claim Returns**: Withdraw earnings using `claim-rewards`
5. **Trade Shares**: Transfer ownership via `transfer-shares`

### Example Workflow

```clarity
;; Create a solar project (contract owner only)
(create-project 
  "Sunny Valley Solar Farm" 
  "California, USA" 
  u1000000  ;; 1MW capacity
  u100000   ;; 0.1 STX per share
  u10000    ;; 10,000 total shares
  u50       ;; Energy rate
)

;; Purchase 100 shares in project 1
(purchase-shares u1 u100)

;; Record energy production (project owner only)
(record-energy-production u1 u1 u50000 u5000000)

;; Calculate and claim rewards
(calculate-rewards tx-sender u1)
(claim-rewards u1)
```

## Economic Model

- **Platform Fee**: 2.5% of revenue goes to platform maintenance
- **Minimum Investment**: 1 STX to participate
- **Reward Distribution**: Based on proportional ownership of project shares
- **Share Pricing**: Set individually for each project by creators

## Security Features

- **Access Control**: Owner-only functions for sensitive operations
- **Input Validation**: Comprehensive parameter checking
- **Error Handling**: Detailed error codes for troubleshooting
- **Balance Verification**: Ensures sufficient funds before transactions

## Error Codes

- `u100`: Owner only operation
- `u101`: Project/resource not found
- `u102`: Unauthorized access
- `u103`: Invalid amount specified
- `u104`: Project is inactive
- `u105`: Insufficient balance
- `u106`: Transfer failed
- `u107`: Project already exists
- `u108`: Invalid parameters
- `u109`: No rewards available
